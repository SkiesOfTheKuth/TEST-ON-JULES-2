# Migrations Guide

This project currently has no database schema. Use this template when introducing persistent storage or API evolutions.

## Template

### Summary
- **Change ID:** `YYYYMMDD-<short-name>`
- **Author:** Name
- **Date:** YYYY-MM-DD

### Context
Explain why the change is needed and any dependencies.

### Plan
1. Describe the steps to apply the change safely.
2. Provide rollback strategy.
3. Note expected user impact.

### Verification
- Commands to validate the migration locally and in staging.
- Metrics or logs to monitor post-deploy.

### Rollback
- Explicit commands or procedures to revert the change.
