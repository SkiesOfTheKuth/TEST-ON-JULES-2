# OPERATIONS

## Runbooks

### 1. Static Host Deployment
1. Run `npm install`.
2. Run `npm run lint` and `npm test`; fix issues.
3. Upload repository contents (or built bundle if using a framework) to static host/CDN.
4. Invalidate CDN cache (if applicable) to propagate updates.

### 2. Incident: Calculations Returning "Error"
1. Reproduce locally with the reported expression.
2. Check browser console/network logs for stack traces.
3. Run `npm test` to ensure expression sanitisation passes.
4. Review `src/lib/calculator.js` for unsupported operators; extend regex/logic if needed.
5. Document resolution in incident log.

### 3. Dark Mode Not Persisting
1. Confirm browser allows `localStorage` (private mode restrictions).
2. Inspect `localStorage.getItem('calculator-theme')` in devtools.
3. Clear storage and retest; if bug persists, file ticket with reproduction steps.

## Monitoring
- Client-side logging: console errors generated by calculation exceptions.
- For production, integrate a privacy-aware client telemetry solution (e.g., Sentry with IP anonymisation) or OSS alternative (OpenTelemetry collector) to capture error rates.

## Backups & Disaster Recovery
- Application code stored in source control (Git) with remote backups.
- Calculator history is client-side only; no server persistence required.

## On-Call Expectations
- Severity 1: calculator unavailable or incorrect results → respond within 1 business hour.
- Severity 2: ancillary feature failure (theme/history) → respond within 4 business hours.
